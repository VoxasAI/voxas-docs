## When to use this

Use this guide after install and permissions are complete.

## User flow

1. Open the sessions list.
2. Create a new session.
3. Start interacting in the session room.

<Note>Image placeholder: Session room UI after creation.</Note>

## How to do it

### Step 1: Open sessions

From the main app shell, navigate to Sessions.

### Step 2: Create a session

Use the create action in the sessions view, choose your session context, and start.

### Step 3: Begin your workflow

In the session room, ask your first prompt and confirm responses are streaming normally.

## What success looks like

- A new session appears in your sessions list.
- The session room loads and accepts your prompts.

## What could go wrong

### Issue: Session creation fails

- **Why it happens:** Network or authentication issue.
- **How to fix:** Re-authenticate, then retry from the sessions list.

### Issue: Session loads but responses do not stream

- **Why it happens:** Connection issue or missing permissions.
- **How to fix:** Verify permissions, then retry. If needed, restart Voxas.
